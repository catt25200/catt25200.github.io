Project : Dungeon

Astrubert is a text-based dungeon RPG game with 2D map and is also one of my homework during the first year study.
It set in a city which get abandon after an accident. Player as a robot need to finish the dungeon and find the source of the accident.
It is a very fun experience at that time. Using 2D array to build a reactble map.
To enhance the game's excitement, I have made a machine that restricts player visibility, adding a new challenge and element of surprise to gameplay
Furthermore, a customized upgrade system has been implemented to enhance user resource management through a chip system.

Fishing

Since I want don't want to make river is just another kind of obstacle.

I decide to make player can have some intercation with it.

Normally the option of fishing is hidden from player, until player get closed to the toxic lake.

Fishing require player to buy a fishing hook at the shop first.

Player successful hook something

Player can upgrade, repair, take quest at the Forge.

Player can buy and sell item at the store

Chip System

To add more depth to the game, I decide to make a system to let player customize character ability.

Since player is a robot, it can increase its ability by inserting chip.

For now only have below 3 different kind of hips can be insert.
  • Improve Attack Damage
  • Improve Visible Range
  • Improve Fishing Skill
Player will have 3 slots for chip insert at the start, can expense more slots by finishing missions

Vision are limited during the dunegon

To make this game have more strategic element.

I designed two ways check current state. Casual check and detail check

Casual check will only get the approximate of the armor status (This will not cost 1 move)

Detail check can check all the state of the player (This will cost 1 move)
